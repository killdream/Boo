=== title: Installing subtitle: Getting it up and running created: 2011-07-02 01:51:01 toc: getting started index: 1 === §§ blurb Getting Boo up and running is *easy-peasy*: Just put an script tag at the bottom of your page. ~~~html~~~ ~~~~~~~~~~ Or, if you're running Node.js: ~~~js~~~ var boo = require('/path/to/boo') ~~~~~~~~ §§ /blurb ## Dependencies Boo depends on the recent ECMAScript 5 additions, such as the functional array methods and the prototypical inheritance additions. If you want it to work on old dinosaurs, you can just include the awesome [es5-shim][] library. Otherwise, just make sure the platform supports the following: - [Object.getPrototypeOf][] - [Object.keys][] - [Object.create][] - [Array.prototype.forEach][] ## Resolving conflicts On platforms that don't support CommonJS modules, Boo will take hold of the global `boo`. If you happen to have something else named `boo` in the global scope, you can resolve the conflict by calling `boo.make_local`: ~~~html~~~ ~~~~~~~~~~ ## Special properties Boo uses the `$traits`, `$ctx` and `$super` properties in objects for a range of stuff, so avoid writing to those in objects that are going to be used by the library ;3 [Object.getPrototypeOf]: https://developer.mozilla.org/en/JavaScript/Reference/Global_Objects/Object/GetPrototypeOf [Object.keys]: https://developer.mozilla.org/en/JavaScript/Reference/Global_Objects/Object/keys [Object.create]: https://developer.mozilla.org/en/JavaScript/Reference/Global_Objects/Object/create [Array.prototype.forEach]: https://developer.mozilla.org/en/JavaScript/Reference/Global_Objects/Array/forEach [es5-shim]: https://github.com/kriskowal/es5-shim
