Dependencies

Boo depends on the recent ECMAScript 5 additions, such as the functional array methods and the prototypical inheritance additions. If you want it to work on old dinosaurs, you can just include the awesome es5-shim library.

Otherwise, just make sure the platform supports the following:

  • Object.getPrototypeOf
  • Object.keys
  • Object.create
  • Array.prototype.forEach

Resolving conflicts

Special properties

Boo uses the $traits, $ctx and $super properties in objects for a range of stuff, so avoid writing to those in objects that are going to be used by the library ;3
